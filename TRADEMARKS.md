# Trademark and Name-Usage Policy

This document is the DedupCommando project's own policy on how its name — and the names of
others — are used. It is **not legal advice and is not a trademark clearance**. The names
mentioned here are used as project and product names; this policy does not assert any
registered trademark.

DedupCommando is free software released under the [Apache-2.0 license](LICENSE). Nothing in
this policy limits any right that license grants you to use, modify, and redistribute the
**software**. The notes below concern the project's **name** only.

## The project's name

- **DedupCommando** is the name of this project and its brand. Refer to it by the full,
  single composite word — "DedupCommando", written as one word with the two capitals. Please
  don't split it ("Dedup Commando") or shorten it to "Commando".
- The installable package, command, and binary are all named **`dedcom`** (for example,
  `apt install dedcom`). The name `dedupcommando` is also reserved as a defensive alias.
- **`dedcom`** is a short, convenient package and command name — **not a brand**. Please don't
  use "dedcom" on its own to name or market the project; the brand is always the full name
  **DedupCommando**.

## Using the name — what's fine

You may use the name **descriptively and nominatively**: to state truthfully that your work
uses, integrates with, supports, or is compatible with DedupCommando, and to write articles,
give talks, and produce documentation about it. References like "works with DedupCommando" or
"a guide to DedupCommando" are welcome and need no permission.

## Using the name — please don't

- Don't use the name, or a confusingly similar one, for a **fork, derivative, or unrelated
  product** in a way that implies it is the official project or is endorsed by it. Give your
  fork a distinct name and state the relationship plainly (for example, "a fork of
  DedupCommando").
- Don't use the name in a way that suggests **affiliation, sponsorship, or endorsement** that
  doesn't exist.
- Don't register the name, confusingly similar names, or matching domains or account handles
  as your own.

## Other names and non-affiliation

DedupCommando is an independent project. It is not affiliated with, sponsored by, or endorsed
by any of the parties below; their names appear only to describe interoperability or for
accurate reference, and each name belongs to its owner.

- "Proxmox" is a trademark of Proxmox Server Solutions GmbH; this project is independent and
  not affiliated with or endorsed by them.
- "ZFS" / "OpenZFS" belong to their respective owners.
- DedupCommando is a single composite name and is not affiliated with any similarly named
  product — including other products that contain the word "Commando", such as the "Commando
  VM" security distribution (Mandiant).

## Questions

For anything not covered here, or for a use you're unsure about, please open an issue or a
discussion in the project repository.
